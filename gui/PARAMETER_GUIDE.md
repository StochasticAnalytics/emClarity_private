# emClarity GUI with Parameter Configuration

This enhanced GUI provides a modern tabbed interface for working with emClarity, with comprehensive parameter configuration capabilities.

## New Features

### 🎛️ Parameter Configuration Panels
- **Split Tab Layout**: Each tab now has commands on the left (30%) and parameter configuration on the right (70%)
- **Enhanced Parameter Widgets**: Support for vectors, choices, file selection, and type validation
- **Rich Tooltips**: Detailed tooltips showing parameter descriptions, ranges, units, and default values
- **Real-time Validation**: Parameter validation with error reporting

### 📁 Parameter File Management
- **Auto-generation**: Create parameter files automatically with datetime stamps
- **Example Loading**: Load example parameters based on `gen_param.m`
- **Export All**: Export parameters from all tabs to a single `.m` file
- **Per-tab Export**: Save parameters for individual functional areas

### 🎨 UI Improvements
- **Keep on Top**: Toggle to keep GUI window on top for editing convenience
- **Font Size Control**: GUI-wide font size adjustment (8-24pt)
- **Soft Blue Theme**: Modern styling with rounded corners and gradients
- **Responsive Layout**: Resizable split panes for optimal workflow

## Parameter Categories

The GUI organizes parameters into functional tabs:

1. **System**: Project configuration, hardware settings (GPUs, CPUs)
2. **CTF**: Microscope parameters, defocus estimation, CTF correction
3. **Alignment**: Particle alignment, mask settings, angular search
4. **Template Search**: Template matching parameters and thresholds
5. **Processing**: Particle properties, classification, PCA settings
6. **Reconstruction**: FSC parameters, reconstruction settings
7. **Utilities**: Dose calculation, fiducial settings, tilt parameters

## Usage

### Running the GUI
```bash
cd /sa_shared/git/emClarity
source .venv/bin/activate
cd gui
python main.py
```

### Testing Parameter System
```bash
python test_parameters.py
```

### Loading Example Parameters
1. Run the GUI
2. Go to **Parameters → Load Example Parameters**
3. Parameters based on `gen_param.m` will be loaded across all tabs

### Creating Parameter Files
1. Configure parameters in the relevant tabs
2. Click "Save Parameter File" in any tab, or
3. Use **Parameters → Export All Parameters** for a complete file

### Parameter File Format
Generated files follow emClarity format:
```matlab
% emClarity parameter file generated on 2025-08-26 11:29:09
% Job type: gui_export
% Auto-generated by emClarity GUI

% System parameters
%==================================================
% Project name for metadata structure
subTomoMeta=my_project

% Number of GPUs to use for computation
nGPUs=4

% Vector parameters use MATLAB array format
particleRadius=[180,180,150]
```

## Parameter Details

### Required Parameters ⚠️
- `subTomoMeta`: Project name
- `nGPUs`: Number of GPUs
- `nCpuCores`: CPU cores
- `PIXEL_SIZE`: Pixel size in meters
- `Cs`: Spherical aberration
- `VOLTAGE`: Accelerating voltage
- `AMPCONT`: Amplitude contrast
- `symmetry`: Particle symmetry

### Vector Parameters
Several parameters accept vector inputs (e.g., `[x,y,z]`):
- `particleRadius`: Particle dimensions
- `Ali_mRadius`: Alignment mask radius
- `whitenPS`: Whitening parameters
- `Tmp_bandpass`: Bandpass filter settings

### Choice Parameters
Dropdown selections for:
- `symmetry`: C1, C2, ..., C12, O, I
- `Ali_mType`: cylinder, sphere, rectangle
- `startingDirection`: pos, neg
- `fastScratchDisk`: ram, /tmp, /scratch

## Integration with emClarity

When running commands through the GUI:
1. Current parameter values are collected from all tabs
2. A temporary parameter file is created automatically
3. The parameter file is passed to the emClarity command
4. Parameter files are timestamped for tracking

## Customization

### Adding New Parameters
Edit `parameters.py` to add new parameters to any category:

```python
Parameter(
    'my_param', 'My Parameter', 'float',
    'Description of what this parameter does',
    default=1.0, min_value=0.1, max_value=10.0,
    unit='Angstrom', required=False
)
```

### Parameter Types
- `string`: Text input
- `int`: Integer spinbox with range validation
- `float`: Decimal spinbox with range validation
- `bool`: Checkbox
- `choice`: Dropdown selection
- `vector`: Multiple numeric inputs
- `file`: File path with browse button

### Validation
Parameters are validated for:
- Required field presence
- Type correctness (int/float/string)
- Range compliance (min/max values)
- Choice validity (dropdown options)
- Vector size (for array parameters)

## Tips

1. **Parameter Persistence**: Use "Export All Parameters" regularly to save your configurations
2. **Validation Feedback**: Watch for validation errors when required parameters are missing
3. **Tooltips**: Hover over any parameter for detailed information
4. **Keep on Top**: Use the toggle when editing parameter files alongside the GUI
5. **Font Size**: Adjust font size in View menu for better readability
6. **Example Parameters**: Start with "Load Example Parameters" as a baseline configuration
